- MASTER THESIS

Evaluating the modernity of the UI in the wake of accessibility. Developing accessible, fresh and new visual design, using Google Material Design components.

TOOLS

Research: Utilia Validator, WAVE Validator, DR. Travis Checklist
Design: Figma, Adobe Illustrator

MY ROLE

Analyze accessible literature, publications and materials in the studied area. Deliver the evaluated accessible website design, basing on literature and the conducted quantitative, qualitative and comparative researches.

GOAL -

The main goal of my research paper was to present what are the main goals of making systems, applications and websites accessible for everyone, especially for disabled people. Furthermore, the additional aim was to enable a knowledgeable reply to the question if the website can be both modern and accessible.

The want to introduce the process of the web accessibility evaluation and draw attention to techniques and approaches, that are best to focus on when enhancing a website with an available, and attractive user interface would help designers, developers and business owners to better evaluate their websites.

CONCLUSIONS -

Accessibility is a big topic, around 15% of today’s population is affected by some kind of disability.  

User interface design plays a crucial role in software development and maintenance.

Users use a system through an interface, which lets them access the information that they are looking for.

My list of accessibility guidelines that should be followed to introduce accessible and modern interfaces.

It is worth to use official accessibility standards guide - I recommend using WCAG 2.1 guidelines.

List of valuable validators, which helps designers and developers to evaluate their websites and applications.

When designer designs from scratch it is easier to create an accessible interface. However, in some cases, a business wants to adjust a current interface to accessibility rules. Then, it is good to use assistive technologies to make this transformation easier.

Even if a website has a positive result during automatic online validation, it does not mean that it is fully accessible and meets A, AA, or AAA level in WCAG 2.0 or WCAG 2.1 guiding principles. 

Adding accessibility requirements to UI increases user interface quality and value.

FUTURE -

I had to limit my focus on the user interface and user experience design theory and concepts rather than the technical aspects of the interface development. The scope of this study was restricted to the one static proposition of new website design. Progress has been made in Razem sprawniej website development, but many important research questions and technical sustenance needs continue.  

A validation puts emphasis more on visual and usable aspects than the source code of this web page evaluation. The future research should focus also more on technical aspects to make this website more accessible also for assistive technology. Those corrections could be connected with improving alternative texts, getting rid of redundancy and taking care of code hierarchy and transparency. 

Besides visual and source code inspection, it is good to take this website under the screen readers examination. Assistive technologies are used by people with accessibilities, that is why the website should, as much as possible, suit to those kinds of assistants. Starting from preinstalled text to speech software to web text-to-speech converters. The good practice is to also evaluate this website using the Voluntary Product Accessibility Template (VPAT). VPAT is free of charge reporting format. It is developed to support consumers and vendors in recognizing accessibility features in products and services.
